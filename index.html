BEM Convention

All of my projects from this point onwards will be following the BEM convention.

BEM is a convention that states how classes should be named to avoid overlap and keep code clean and digestible.

Put simply, it keeps everything organised for the current developer or future developers to pick back up and work on in the future. As well as preventing a lot of errors and making troubleshooting significantly easier.
